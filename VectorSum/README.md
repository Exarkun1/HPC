# Задание
### Перемножение матриц
**Задача:** реализовать алгоритм сложения элементов вектора  
**Язык:** C++ или Python  
**Входные данные:** Вектор размером 1 000..1 000 000 значений.  
**Выходные данные:** сумма элементов вектора + время вычисления  
Реализация должна содержать 2 функции сложения элементов вектора: на CPU и на
GPU с применением CUDA.

# Описание проделанной работы
Данная лабораторная работа была выполнена на языке С++.

**CPU:** в функции `vector_sum_cpu` реализовано последовательное суммирование.

**GPU:** в функции `vector_sum_gpu` реализовано параллельное суммирование за счет вызова ядра `vector_sum_kernel`. В ядре реализована параллельная редукция, каждый блок суммирует элементы в shared memory, затем блоки редуцируются. В конце первый поток блока вызвет атомарную операцию сложения результата с частичной суммой, полученной текущим блоком.

Как видно из таблицы при малых размерах вектора GPU проигрывает CPU во времени вычислений, но с ростом размера GPU начинает вычислять вектор быстрее.

В таблице приведены результаты вычислений как с учетом копирования массивов на девайс и обратно, так и без учета. Так как вектор имеет сравнительно небольшие размеры, копирование сильно влияет на ускорение, при дальнейшем увеличении вектора оно будет всё меньше влиять на ускорение.

Таким образом, в ходе работы была проведена оценка производительности суммирования элементов вектора на CPU и GPU. Для малых размеров CPU показал хорошее время выполнения из-за меньших накладных расходов. Однако с увеличением размера данных GPU начинает выигрывает за счёт параллельной обработки, что подтверждает эффективность использования GPU для обработки больших объёмов данных.
